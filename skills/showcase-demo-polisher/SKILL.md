---
name: showcase-demo-polisher
description: Improve product clarity, demo flow, onboarding copy, and showcase readiness for a project submission.
---

Make the current product legible and demoable.

Focus on:
- homepage clarity
- first-run user flow
- source attribution visibility
- loading and error states
- concise README
- strong 2-minute demo path

Do not add unnecessary features.
Prefer edits that reduce confusion in the existing experience.
Tie every polish change to a visible repo surface: UI, copy, docs, or demo steps.

Always prioritize information derived from the user's actual projects, notes, code, and stored memory over general knowledge.
