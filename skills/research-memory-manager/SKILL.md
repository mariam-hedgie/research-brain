---
name: research-memory-manager
description: Convert work sessions, coding changes, and notes into compact structured project memory for Research Brain.
---

Convert raw activity into durable project memory.

Always do the following:
1. Identify the active project.
2. Extract only high-value facts:
   - goal changes
   - blockers
   - completed work
   - failed attempts
   - next planned action
   - important files, papers, or experiments touched
3. Separate output into:
   - canonical memory: stable project facts
   - episodic memory: this specific work session
   - action memory: next step and prerequisites
4. Keep summaries concise and specific.
5. Omit filler, advice, and anything not grounded in the session.
6. Prefer structured JSON output when asked.

When summarizing a coding session, include:
- files changed
- purpose of change
- validation done
- unresolved issues
- recommended next task

Always prioritize information derived from the user's actual projects, notes, code, and stored memory over general knowledge.
